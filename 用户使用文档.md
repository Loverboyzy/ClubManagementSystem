## 1. 系统概述

社团管理系统是一套用于高等院校或组织中社团日常管理的 Web 应用系统。系统支持**系统管理员**、**社团管理员（团长）** 和**普通成员**三种角色，涵盖社团的创建、成员入团申请与审批、活动发布与报名、缴费记录、通知公告等完整业务流程。

### 核心功能一览

| 功能模块 | 说明 |
|----------|------|
| 用户认证 | 登录 / 退出 / 个人信息修改 / 密码修改 |
| 用户管理 | 用户增删改查，支持分页与条件筛选 |
| 社团类型 | 社团分类管理（如科技类、文学类等） |
| 社团管理 | 创建社团、编辑信息、删除社团 |
| 成员管理 | 社团成员列表、入团与退出 |
| 入团申请 | 提交申请、审批（通过/拒绝） |
| 活动管理 | 发布活动、编辑、删除 |
| 活动报名 | 报名活动、查看报名记录、防重复报名 |
| 通知公告 | 发布通知、按角色推送 |
| 缴费管理 | 记录缴费信息、按社团/用户查询 |

## 3. 环境要求与部署

### 3.1 环境要求

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 1.8+ | Java 运行环境 |
| MySQL | 8.0+ | 数据库（端口 3306） |
| Maven | 3.3+ | 后端构建工具 |
| Node.js | 12+ | 前端运行环境（如使用前端） |
| npm | 6+ | 前端包管理 |

### 3.2 数据库准备

**步骤一：创建数据库**

```sql
CREATE DATABASE association_manager
  CHARACTER SET utf8
  COLLATE utf8_general_ci;
```

**步骤二：导入建表脚本**

```bash
mysql -u root -p association_manager < 数据库脚本.sql
```

> 数据库脚本位于项目根目录的 `数据库脚本.sql` 文件中，包含所有建表语句和初始测试数据。

**步骤三：配置数据库连接**

编辑 `src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    username: root           # 改为你的数据库用户名
    password: 123456         # 改为你的数据库密码
    url: jdbc:mysql://localhost:3306/association_manager?...
```

### 3.3 启动后端服务

**方式一：IDE 启动**（推荐）

在 IntelliJ IDEA 或 Eclipse 中直接运行 `AssociationManagerApplication.java` 中的 `main` 方法。

**方式二：Maven 命令行启动**

```bash
cd AssociationManagerApi
mvn clean package -DskipTests
mvn spring-boot:run
# 或
java -jar target/AssociationManagerApi-1.0.0.jar
```

**启动成功标志**：控制台输出 `Started AssociationManagerApplication in X seconds`，后端默认监听 **9211** 端口。

### 3.4 启动前端服务

```bash
cd AssociationManagerVue
npm install          # 首次运行需安装依赖
npm run serve        # 启动开发服务器
```

前端默认监听 **9212** 端口，浏览器访问 `http://localhost:9212`。

### 3.5 服务端口汇总

| 服务 | 地址 | 端口 |
|------|------|------|
| MySQL 数据库 | localhost | 3306 |
| 后端 API | localhost | 9211 |
| 前端页面 | localhost | 9212 |
| API 基础路径 | `http://localhost:9211/association` | - |

## 4. 数据库设计

### 4.1 ER 图概览

系统共有 **9 张数据库表**，关系如下：

```
users (用户)
  ├── members (成员关系) ──→ teams (社团) ──→ team_types (社团类型)
  ├── apply_logs (入团申请) ──→ teams
  ├── active_logs (活动报名) ──→ activities (活动) ──→ teams
  ├── pay_logs (缴费记录) ──→ teams
  └── notices (通知) ──→ teams (可选)
```

## 5. 用户角色与权限

系统设计了**三级角色体系**，不同角色拥有不同的数据访问权限：

| type | 角色名称 | 权限范围 |
|------|----------|----------|
| **0** | 系统管理员 | 最高权限，可查看和管理系统中**所有数据**（所有用户、所有社团、所有记录） |
| **1** | 社团管理员（团长） | 可管理**自己创建的社团**相关的数据（成员审批、活动发布、通知发布、缴费记录） |
| **2** | 普通成员 | 仅可查看和操作**自己的数据**（个人信息、入团申请、活动报名、缴费记录） |

### 预置账号

| 账号 | 密码 | 角色 | type |
|------|------|------|------|
| `superadmin` | `123456` | 系统管理员 | 0 |
| `admin` | `123456` | 社团管理员（团长） | 1 |
| `user` | `123456` | 普通成员 | 2 |

## 6. 功能模块详解

### 6.1 用户登录与认证

系统使用 **Token 认证机制**：用户登录成功后，后端生成 UUID Token 并存入 Ehcache 缓存，前端将 Token 存储在 `sessionStorage` 中，后续请求通过 `token` 参数传递身份凭证。

#### 登录流程

```
用户输入账号密码 → POST /login
    ├── 用户名不存在 → 返回错误提示
    ├── 密码错误 → 返回错误提示
    └── 验证通过 → 生成 token，存入缓存，返回 token 给前端
```

#### 相关接口

| 方法 | 路径 | 参数 | 说明 |
|------|------|------|------|
| POST | `/login` | userName, passWord | 用户登录，返回 token |
| GET | `/exit` | token | 退出登录，清除缓存 |
| GET | `/info` | token | 获取当前登录用户信息 |
| POST | `/info` | 用户信息字段 | 修改个人信息 |
| GET | `/checkPwd` | token, oldPwd | 验证原密码是否正确 |
| POST | `/pwd` | token, password | 修改密码 |
| GET | `/sys/notices` | token | 获取当前用户可见的通知列表 |

### 6.2 用户管理

> **权限**：系统管理员可操作所有用户；社团管理员和普通成员仅可查看/修改自己的信息。

#### 功能操作

- **新增用户**：管理员可直接添加用户（设置账号、密码、角色等）
- **修改用户**：编辑用户的基本信息
- **删除用户**：删除前系统会检查该用户是否存在关联数据（社团、成员、活动等）
- **分页查询**：支持按 `userName`（账号）、`name`（姓名）、`phone`（电话）进行筛选

#### 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/users/page` | 分页查询用户列表 |
| POST | `/users/add` | 添加用户 |
| POST | `/users/upd` | 修改用户信息 |
| POST | `/users/del` | 删除用户（传 id） |

### 6.3 社团类型管理

> **权限**：管理员可进行增删改查；普通用户可查看。

社团类型用于对社团进行分类，如：**科技类、文学类、艺术类、体育类、公益类**等。

- **新增类型**：设置类型名称
- **修改类型**：修改类型名称
- **删除类型**：删除前会检查该类型下是否有关联的社团，有关联则不可删除

#### 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/teamTypes/all` | 获取全部社团类型 |
| GET | `/teamTypes/page` | 分页查询类型列表 |
| POST | `/teamTypes/add` | 添加社团类型 |
| POST | `/teamTypes/upd` | 修改社团类型 |
| POST | `/teamTypes/del` | 删除社团类型 |

### 6.4 社团管理

> **权限**：系统管理员可管理所有社团；社团管理员可管理自己创建的社团；普通成员仅可查看。

#### 功能操作

- **创建社团**：填写社团名称、选择类型、指定团长
- **编辑社团**：修改社团信息
- **删除社团**：删除社团及其关联数据
- **查看社团**：支持按社团名称和类型筛选

#### 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/teams/all` | 获取全部社团列表 |
| GET | `/teams/page` | 分页查询社团（支持 name、typeId 筛选） |
| POST | `/teams/add` | 创建社团 |
| POST | `/teams/upd` | 修改社团信息 |
| POST | `/teams/del` | 删除社团 |

### 6.5 成员管理

> **权限**：管理员可查看所有成员记录；社团管理员可查看自己社团的成员；普通成员查看自己的记录。

成员表记录用户与社团的关联关系。当入团申请被通过后，系统自动将用户加入该社团的成员列表。

- **添加成员**：直接将用户添加为某社团成员
- **移除成员**：将用户从社团中移除（**团长不可被移除**）
- **分页查询**：按权限范围展示成员数据

#### 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/members/page` | 分页查询成员列表 |
| POST | `/members/add` | 添加成员 |
| POST | `/members/upd` | 修改成员信息 |
| POST | `/members/del` | 删除成员（团长不可删除） |

### 6.6 入团申请管理

> **权限**：普通成员可提交申请和查看自己的申请；社团管理员可审批自己社团的申请；系统管理员可查看所有。

#### 申请流程

```
普通成员提交入团申请 → POST /applyLogs/add（status=0 待审批）
        │
        ▼
社团管理员查看申请列表 → GET /applyLogs/page
        │
        ├── 通过 → status=1，同时自动加入 members 表
        └── 拒绝 → status=2，仅更新状态
```

#### 业务规则

- 同一用户对同一社团**不可重复申请**（系统会检测）
- 申请状态：**0=待审批，1=已通过，2=已拒绝**
- 审批通过后，系统**自动**将该用户添加为社团成员

#### 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/applyLogs/page` | 分页查询申请记录 |
| POST | `/applyLogs/add` | 提交入团申请 |
| POST | `/applyLogs/upd` | 审批申请（修改 status） |
| POST | `/applyLogs/del` | 删除申请记录 |

### 6.7 活动管理

> **权限**：社团管理员可发布和管理自己社团的活动；普通成员可查看活动。

#### 功能操作

- **发布活动**：填写活动名称、概述、详情、要求、活动时间
- **编辑活动**：修改活动信息
- **删除活动**：删除活动
- **分页查询**：按权限范围展示活动

#### 活动字段

| 字段 | 说明 |
|------|------|
| 活动名称 | 最多 60 字 |
| 活动概述 | 简要描述活动内容 |
| 活动详情 | 详细活动说明 |
| 活动要求 | 参与活动需要满足的条件 |
| 报名人数 | 系统自动更新（有人报名即+1） |
| 活动时间 | 活动举办的具体时间 |

#### 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/activities/page` | 分页查询活动列表 |
| POST | `/activities/add` | 发布活动 |
| POST | `/activities/upd` | 修改活动信息 |
| POST | `/activities/del` | 删除活动 |

### 6.8 活动报名管理

> **权限**：普通成员可报名和查看自己的报名记录；社团管理员可查看自己社团活动的报名情况。

#### 报名流程

```
成员选择活动 → POST /activeLogs/add
        │
        ├── 检测是否重复报名 → 是 → 返回错误提示
        └── 未重复 → 创建报名记录，活动报名人数+1
```

#### 业务规则

- 同一用户对同一活动**不可重复报名**
- 报名成功后，对应活动的 `total` 字段自动 +1

#### 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/activeLogs/list` | 获取指定活动的报名列表 |
| POST | `/activeLogs/add` | 报名活动 |
| POST | `/activeLogs/upd` | 修改报名记录 |
| POST | `/activeLogs/del` | 删除报名记录 |

### 6.9 通知公告管理

> **权限**：系统管理员可发布系统级通知（所有用户可见）；社团管理员可发布社团级通知（该社团成员可见）；普通成员仅可查看与自己相关的通知。

#### 通知类型

| 类型 | team_id | 可见范围 |
|------|---------|----------|
| 系统通知 | NULL | 所有用户 |
| 社团通知 | 具体社团 ID | 该社团成员 |

#### 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/notices/page` | 分页查询通知列表 |
| POST | `/notices/add` | 发布通知 |
| POST | `/notices/upd` | 修改通知 |
| POST | `/notices/del` | 删除通知 |

### 6.10 缴费管理

> **权限**：系统管理员可查看所有缴费记录；社团管理员可查看自己社团的缴费记录；普通成员可查看自己的缴费记录。

```json

| code | 含义 | 说明 |

```

| 字段 | 类型 | 说明 |
|------|------|------|
| pageIndex | Long | 当前页码（从 1 开始） |
| pageSize | Long | 每页记录数 |
| pageTotal | Long | 总页数 |
| count | Long | 总记录数 |
| data | List | 当前页数据列表 |

## 9. 常见问题 (FAQ)

### Q1：启动后端报 "Failed to obtain JDBC Connection"

**原因**：数据库连接配置有误。

**解决**：
1. 确认 MySQL 服务已启动（端口 3306）
2. 检查 `application.yml` 中的数据库用户名和密码是否正确
3. 确认数据库 `association_manager` 已创建

### Q2：前端访问后端报 CORS 跨域错误

**错误信息**：`Access to XMLHttpRequest has been blocked by CORS policy`

**解决**：确认后端 `AssociationManagerApplication.java` 中的 `corsConfigurer()` 方法存在且配置正确。如果使用生产环境部署，建议配置 Nginx 反向代理。

### Q3：登录后提示 "登录信息不存在，请重新登录"

**原因**：Token 过期或缓存丢失。

**解决**：
1. Token 存储在 Ehcache 中，后端重启后缓存会清空
2. 前端 Token 存储在 sessionStorage 中，关闭浏览器标签页后会清空
3. 重新登录即可获取新的 Token

### Q4：为什么删除用户提示失败？

**原因**：该用户存在关联数据（如：是该社团的团长、有成员记录、有活动报名记录等）。

**解决**：先删除该用户的所有关联数据，再删除用户。

### Q5：为什么删除社团类型提示失败？

**原因**：该类型下存在关联的社团。

**解决**：先将属于该类型的所有社团转移到其他类型或删除，再删除类型。

### Q6：同一用户能否重复报名同一活动？

**不能**。系统后台会检测并阻止重复报名，返回警告信息。

### Q7：同一用户能否重复申请加入同一社团？

**不能**。系统后台会检测并阻止重复申请。

### Q8：能否将团长从社团成员中移除？

**不能**。系统会检测当前要删除的成员是否为社团团长，团长不可被移除。

### Q9：通知的区别是什么？

- **系统通知**（team_id 为 NULL）：所有登录用户可见
- **社团通知**（team_id 指向具体社团）：仅该社团成员可见
- 系统管理员发布的通知是系统级通知；社团管理员发布的是社团级通知

### Q10：如何重置密码？

普通用户可登录后在个人信息页面修改密码（需验证原密码）。管理员可以在用户管理模块直接修改任意用户的密码。
